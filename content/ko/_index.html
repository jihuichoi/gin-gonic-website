{{< /blocks/cover >}} {{% blocks/lead color="white" %}} **Gin 이 뭔가요?** Gin 은 고언어(Golang)으로 제작한 웹 프레임워크입니다. Gin 은 최대 40배 빠른 마티니 스타일의 API 를 제공합니다. 성능과 생산을 모두 만족하는 웹 프레임워크를 찾는다면 Gin 이 정답입니다. {{% /blocks/lead %}} {{< blocks/section color="light" >}} {{% blocks/feature icon="fa fa-tachometer-alt" title="빠름" %}} Radix 트리 기반의 라우팅, 적은 양의 메모리 사용. reflection 사용 안함. 예측 가능한 API 성능 {{% /blocks/feature %}} {{% blocks/feature icon="fa fa-bars" title="미들웨어 지원" %}} HTTP 요청을 미들웨어와 최종 동작의 연계에 맞추어 처리합니다. 예: 로거, 인증, GZIP 압축, DB에 메시지 저장 등 {{% /blocks/feature %}} {{% blocks/feature icon="fa fa-life-ring" title="충돌 없음" %}} Gin 은 HTTP 요청 중 발생한 패닉을 자동으로 리버커하므로, 서버가 다운되는 일이 없습니다. 패닉의 내용을 Sentry 와 같은 트래커로 보낼 수도 있습니다. {{% /blocks/feature %}} {{< /blocks/section >}} {{< blocks/section color="white" >}} {{% blocks/feature icon="fa fa-check-circle" title="JSON 검증" %}} Gin 은 JSON 요청 내용을 파싱하고 검증할 수 있습니다. 예를 들어 필요한 값의 존재 여부를 확인할 수 있습니다. {{% /blocks/feature %}} {{% blocks/feature icon="fa fa-users-cog" title="라우트 그룹" %}} 라우트 그룹을 이용해 라우트를 더 정교하게 정리할 수 있습니다. 예를 들어 인증이 필요한 경우와 아닌 경우에 따라 분리할 수 있습니다. 또한 하위 그룹을 성능 저하없이 무한대로 만들 수 있습니다. {{% /blocks/feature %}} {{% blocks/feature icon="fa fa-briefcase" title="오류 관리" %}} HTTP 요청 중에 발생한 모든 오류를 간편하게 수집할 수 있습니다. 이렇게 오류를 모아 미들웨어를 통해 로그파일이나 데이터베이스로 전달할 수 있습니다. {{% /blocks/feature %}} {{< /blocks/section >}} {{< blocks/section color="info" >}} {{% blocks/feature icon="fa fa-images" title="내장 렌더링" %}} JSON, XML, HTML 로 렌더링할 수 있는 간편한 API를 제공합니다. {{% /blocks/feature %}} {{% blocks/feature icon="fa fa-code" title="확장성" %}} 손쉽게 미들웨어를 생성하고, 예제 코드를 이용해 확인해 볼 수 있습니다. {{% /blocks/feature %}} {{< /blocks/section >}}
